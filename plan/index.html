The Verbex Compiler: Execution Master Plan

This is your step-by-step roadmap to building the compiler in C++, generating LLVM IR, and targeting the Cortex-M0.

Phase 1: Architecture & Foundation (Days 1-2) * Action: Initialize the C++ project using CMake. Link LLVM (version 14+ recommended) and ensure the ARM target libraries are included in your CMakeLists.txt. * Action: Define your core data structures. Create a Token structure that holds the token type, the string value, and—crucially for your Rust-style errors—the line_number and column_number.

Phase 2: The Lexer & Error Tracking (Days 3-4) * Action: Write the Lexer class. It reads the raw .vbx text file. * Action: Implement the keyword matching for and, or, not, print, int, if, else, while. * Action: Build the location tracker. Every time the Lexer consumes a newline (\n), increment the line counter and reset the column counter.

Phase 3: The Parser & AST Visualizer (Days 5-7) * Action: Define your Abstract Syntax Tree (AST) node classes in C++ (e.g., VariableDeclAST, BinaryOpAST, IfExprAST). * Action: Write the Recursive Descent Parser to build the tree from your Lexer's tokens. * Action: The DX Flex: Implement a dump() method on every AST node. Add a --dump-ast flag to your main int main(int argc, char** argv) function. Before generating any LLVM code, ensure you can print a beautiful, indented tree of your parsed code to the terminal.

Phase 4: LLVM IR Generation & The Print Feature (Days 8-10) * Action: Initialize llvm::LLVMContext, llvm::Module, and llvm::IRBuilder. * Action: Add a codegen() method to all AST nodes to traverse the tree and generate standard LLVM IR. * Action: The Native Print Flex: In your module initialization, use LLVM's Function::Create to declare the standard C printf function. When your AST encounters a print x; statement, generate an LLVM CallInst to pass x to printf.

Phase 5: Cortex-M0 Target Emission (Day 11) * Action: Write the back-end hook. Initialize LLVMInitializeARMTargetInfo(), LLVMInitializeARMTarget(), etc. * Action: Set your Module's Target Triple to arm-none-eabi and the CPU to cortex-m0. * Action: Use llvm::legacy::PassManager to emit the final object file (.o).

Phase 6: Testing & Packaging (Day 12-14) * Action: Write 3-5 sample .vbx programs (e.g., a Fibonacci calculator, a simple while loop with a print statement). * Action: Polish the Rust-style error reporting. If the parser fails, fetch the exact line from the source file and print a caret (^) under the tracked column number. * Action: Write a ruthless, professional README detailing the architecture, how to build it via CMake, and why you made your specific design choices.

The Verbex Language Specification (EBNF)

Here is the exact grammar for your parser. It includes the modern logical operators and the built-in print statement.

// ============================================================================
// Verbex Language Specification (EBNF Grammar)
// Target: LLVM IR -> ARM Cortex-M0
// ============================================================================

<program> ::= <declaration_list>

<declaration_list> ::= <declaration> { <declaration> }

<declaration> ::= <variable_declaration> | <function_declaration>

<variable_declaration> ::= "int" <identifier> "=" <expression> ";"
                         | "int" <identifier> ";"

<function_declaration> ::= "int" <identifier> "(" <params> ")" <compound_statement>
                         | "void" <identifier> "(" <params> ")" <compound_statement>

<params> ::= "void" | <param_list> | <empty>

<param_list> ::= <param> { "," <param> }

<param> ::= "int" <identifier>

<compound_statement> ::= "{" <statement_list> "}"

<statement_list> ::= { <statement> }

<statement> ::= <variable_declaration>
              | <expression_statement>
              | <compound_statement>
              | <selection_statement>
              | <iteration_statement>
              | <return_statement>
              | <print_statement>

<expression_statement> ::= [ <expression> ] ";"

<selection_statement> ::= "if" "(" <expression> ")" <statement> [ "else" <statement> ]

<iteration_statement> ::= "while" "(" <expression> ")" <statement>

<return_statement> ::= "return" [ <expression> ] ";"

<print_statement> ::= "print" <expression> ";"

// --- Expressions (Ordered by Precedence) ---

<expression> ::= <identifier> "=" <expression> 
               | <logical_or_expression>

<logical_or_expression> ::= <logical_and_expression> { "or" <logical_and_expression> }

<logical_and_expression> ::= <equality_expression> { "and" <equality_expression> }

<equality_expression> ::= <relational_expression> { <equality_operator> <relational_expression> }

<equality_operator> ::= "==" | "!="

<relational_expression> ::= <additive_expression> { <relational_operator> <additive_expression> }

<relational_operator> ::= "<" | "<=" | ">" | ">="

<additive_expression> ::= <term> { <additive_operator> <term> }

<additive_operator> ::= "+" | "-"

<term> ::= <unary_expression> { <multiplicative_operator> <unary_expression> }

<multiplicative_operator> ::= "*" | "/"

<unary_expression> ::= "not" <unary_expression>
                     | "-" <unary_expression>
                     | <factor>

<factor> ::= "(" <expression> ")" 
           | <identifier> 
           | <number> 
           | <function_call>

<function_call> ::= <identifier> "(" <args> ")"

<args> ::= <arg_list> | <empty>

<arg_list> ::= <expression> { "," <expression> }

// --- Lexical Tokens ---
<identifier> ::= <letter> { <letter> | <digit> | "_" }
<number>     ::= <digit> { <digit> }
<letter>     ::= "a".."z" | "A".."Z"
<digit>      ::= "0".."9"
<empty>      ::=
